# 🎨 Referencias de Diseño - MX WHMCS Modules

Esta carpeta contiene imágenes de referencia para inspiración de diseño descargadas de Pinterest, Dribbble, Behance, etc.

## 📁 Estructura de Carpetas

### `/hero-sections/`
Diseños de hero sections para homepage y páginas principales.

**Elementos a extraer:**
- Layout y composición
- CTAs y su posicionamiento
- Uso de gradientes y colores
- Tipografía y jerarquía
- Imágenes/mockups de producto

**Archivos:** `hero-1.jpg`, `hero-2.jpg`, etc.

---

### `/product-cards/`
Diseños de cards para módulos/productos.

**Elementos a extraer:**
- Diseño de card (borde, sombra, hover)
- Iconografía
- Layout de información (título, descripción, precio)
- Badges y tags
- Estados de hover/active

**Archivos:** `card-1.jpg`, `card-2.jpg`, etc.

---

### `/pricing-tables/`
Diseños de tablas de pricing y comparación.

**Elementos a extraer:**
- Layout de tiers (2, 3, o 4 columnas)
- Highlight del tier recomendado
- Lista de features incluidas
- CTAs en cada tier
- Badges ("Most Popular", "Best Value")

**Archivos:** `pricing-1.jpg`, `pricing-2.jpg`, etc.

---

### `/features-sections/`
Diseños de secciones de features/beneficios.

**Elementos a extraer:**
- Grid layout (2x2, 3x1, 4x1, etc.)
- Iconografía de features
- Combinación de imagen + texto
- Alternancia izquierda-derecha
- Uso de fondos y espaciado

**Archivos:** `features-1.jpg`, `features-2.jpg`, etc.

---

### `/complete-pages/`
Screenshots de páginas completas para referencia general.

**Tipos de páginas:**
- Homepages de SaaS/software
- Páginas de producto
- Marketplaces de plugins/modules
- Pricing pages completas
- About pages

**Archivos:** `homepage-ref-1.jpg`, `modules-page-ref.jpg`, etc.

## 📝 Cómo Usar Estas Referencias

### 1. Selección de Elementos
Para cada componente que vayas a crear:
1. Revisa 3-5 referencias relacionadas
2. Identifica patrones comunes
3. Selecciona los mejores elementos de cada uno
4. Adapta a nuestro Design System (colores, tipografía, spacing)

### 2. No Copiar Directamente
❌ No copiar pixel-perfect
✅ Inspirarse en layout, jerarquía, interacciones
✅ Adaptar a nuestra marca y sistema de diseño
✅ Combinar mejores prácticas de varias referencias

### 3. Documentar Decisiones
Al usar una referencia, documenta:
- Qué imagen inspiró el diseño
- Qué elementos específicos se usaron
- Qué se modificó y por qué

## 🎨 Paleta de Colores (Del Design System)

### Primary Colors
- **Blue Primary**: `#2563eb` (botones, links)
- **Purple Secondary**: `#9333ea` (acentos, premium)

### Semantic Colors
- **Success**: `#22c55e`
- **Warning**: `#f59e0b`
- **Error**: `#ef4444`
- **Info**: `#06b6d4`

### Gradients
- **Primary Gradient**: `linear-gradient(135deg, #2563eb 0%, #9333ea 100%)`
- **Subtle Gradient**: `linear-gradient(to bottom right, #f9fafb 0%, #f3f4f6 100%)`

## 📐 Tipografía

- **Font Family**: Geist Sans (headings), Geist Mono (code)
- **Hierarchy**:
  - H1: `text-5xl` (48px) - `font-extrabold`
  - H2: `text-3xl` (30px) - `font-bold`
  - H3: `text-2xl` (24px) - `font-semibold`
  - Body: `text-base` (16px) - `font-normal`
